# portal-main-journal

一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或网站，仅作为静态 HTML 资源的集中存放与版本管理工具。

## 项目简介

`portal-main-journal` 旨在为多个独立的 HTML 页面提供统一的归档与发布环境。  
每个页面均可视为独立的条目，彼此之间无强制关联，适合用于记录、展示或存档各类前端内容。

## 目录结构

```
portal-main-journal/
├── index.html          # 仓库入口页面（可选）
├── pages/              # 存放所有独立 HTML 页面
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
├── README.md           # 本文件
└── .gitignore          # Git 忽略规则
```

- **pages/**：每个 `.html` 文件代表一个独立的归档页面，可自由添加或删除。
- **assets/**：存放页面间共享的样式、脚本或媒体文件，避免重复引用。
- **index.html**：可选入口文件，用于生成页面列表或导航。

## 页面归档说明

- 每个页面应保持独立性，尽量不依赖仓库外部的资源。
- 页面内容无固定格式，可根据需要自由设计。
- 建议为每个页面添加明确的标题和描述，方便后续检索。
- 如需引用共享资源，请使用相对路径指向 `assets/` 目录。

## 维护说明

- 添加新页面：在 `pages/` 下创建新的 `.html` 文件，并确保其可正常访问。
- 修改已有页面：直接编辑对应文件，提交前请测试页面显示效果。
- 删除页面：移除对应文件，并检查是否还有其他页面引用其内容。
- 建议为每次操作编写简洁的 commit 信息，便于追踪变更。

## 注意事项

- 本仓库不包含任何形式的自动部署配置，如需托管请自行配置。
- 请勿在页面中嵌入未经授权的第三方资源或敏感信息。
- 保持仓库整洁，定期清理无用文件。
